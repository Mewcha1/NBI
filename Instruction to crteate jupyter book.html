Step 2: Create a Jupyter Book#

Initialize the Book: Create a new Jupyter Book structure:Add Content: Navigate to the created folder [name] and add your content: Place Markdown (.md) or Jupyter Notebook (.ipynb) files in the appropriate directories. Update the toc.yml file to define the structure of your book.

#This generates the HTML files in _build/html.
jupyter-book build NBI2

Step 4: Publish to GitHub#

Create a GitHub Repository:Go to GitHub and create a new repository. Do not initialize the repository with a README file (it will conflict with your local files). Then clone to add all the files to the created repository using the following command. NBI-project is the repository created in gitgub

git clone [repository link, eg.  https://github.com/Mewcha1/NBI ]

Step 5: copy all file from the folder in step3 to the folder in step 4 (github repository)#

For better handling of folder structures, use the robocopy command (available by default in Windows):

robocopy NBI2 NBI /E

Step 6: Uplload your html to the repository#

change the directory in the anaconda propmt to the repository folder (NBI-project). use ghp-import tool to publish the html. The ghp-import Python package is a utility designed to quickly deploy static files (like the HTML files generated for your Jupyter Book) to GitHub Pages.

#transferring your built website to a GitHub Pages branch (gh-pages)
pip install ghp-import
#Deploy to GitHub Pages 
ghp-import -n -p -f _build/html

Step 4: Enable GitHub Pages#

  1. Go to the Settings of your GitHub repository.

  2. Scroll down to the Pages section.

  3. Under Source, select gh-pages (or the appropriate branch if you’ve built your book differently).

  4. Save the settings, and GitHub will generate a public URL for your book.
